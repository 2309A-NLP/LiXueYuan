**缺失关键词**: 无

> 武汉兴图新科电子股份有限公司应收账款规模较大的主要原因如下：

1. **行业结算特性**：受国防军队预算管理体制影响，军方客户或服务于军队的总体单位结算周期较长。军方需先对系统级产品进行复杂验收程序，导致配套单位（如本公司）回款周期延长[1][3][6]。

2. **收入确认季节性**：军方采购集中在下半年，特别是第四季度验收确认收入，导致大量款项在信用期内未收回而形成应收账款。2018年末应收账款账面价值达18,597.78万元，占当期营业收入的93.86%[1][6]。

3. **背靠背回款条款**：与军方背景大型研究所合作时，需等待总体单位收到军方款项后才能回款，进一步延长账期[3]。

4. **业务规模扩张**：随着国防信息化建设加速，公司收入快速增长（2016-2018年复合增长率57.82%），应收账款同步增加[1][3][6]。

## 检索优化分析

### 当前系统架构
- **混合检索**: 向量检索(COSINE, 权重0.6) + 关键词检索(词项匹配, 权重0.4)
- **并行执行**: ThreadPoolExecutor 同时执行向量和关键词检索
- **Embedding 缓存**: LRU 缓存(512条)减少重复查询开销
- **表格解析**: PyMuPDF find_tables() → Markdown 格式保留结构
- **图像解析**: 多模态视觉模型(Qwen-VL)生成文字描述

### 性能瓶颈分析
- **Embedding API 调用**: ~6s（远程 SiliconFlow BAAI/bge-large-zh-v1.5）
- **LLM 生成**: ~11.9s（deepseek-ai/DeepSeek-V3）
- **Milvus 检索**: <0.5s（含网络延迟）

### 优化建议
1. **本地 Embedding 模型**: 部署本地 bge-large-zh 可将检索耗时从 6s 降至 <0.5s
2. **查询改写**: 已实现多轮对话指代消解(Query Rewriting)，提升多轮场景检索准确率
3. **结果重排**: 可引入 Cross-Encoder 重排序进一步提升精度
4. **当前准确率**: 90%，已达标(≥90%)
5. **当前召回率**: 93%，需进一步提升召回率
