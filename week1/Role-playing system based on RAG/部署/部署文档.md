# 金融问答系统 Docker 部署文档

## 1. 环境要求

| 依赖 | 最低版本 |
|------|----------|
| Docker | 20.10+ |
| Docker Compose | 2.0+ |
| 磁盘空间 | ≥ 10 GB |
| 内存 | ≥ 4 GB |

## 2. 目录结构

```
├── Dockerfile            # 后端多阶段构建
├── docker-compose.yml    # 编排文件（MySQL + Redis + Milvus + Backend + Nginx）
├── nginx.conf            # Nginx 反向代理配置
├── .dockerignore         # Docker 构建排除列表
├── .env                  # 环境变量（敏感配置）
```

## 3. 快速部署

### 3.1 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 填入必要配置
vim .env
```

关键配置项：

| 变量 | 说明 | 示例 |
|------|------|------|
| `RAG_OPENAI_API_KEY` | LLM API 密钥 | `sk-xxx` |
| `RAG_LLM_MODEL_NAME` | 模型名称 | `deepseek-ai/DeepSeek-V3` |
| `RAG_OPENAI_API_BASE` | API 地址 | `https://api.siliconflow.cn/v1` |
| `MYSQL_ROOT_PASSWORD` | MySQL 密码 | `*` |
| `RAG_JWT_SECRET` | JWT 密钥 | 生产环境必须更换 |

### 3.2 一键启动

```bash
# 构建并启动所有服务
docker-compose up -d --build

# 查看服务状态
docker-compose ps

# 查看后端日志
docker-compose logs -f backend
```

### 3.3 访问地址

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端页面 | http://localhost:5173 | Nginx 代理 |
| 后端 API | http://localhost:8000 | FastAPI |
| API 文档 | http://localhost:8000/docs | Swagger UI |
| Milvus | http://localhost:19530 | 向量数据库 |
| MySQL | localhost:3306 | 关系数据库 |

## 4. 数据持久化

| Docker Volume | 宿主机目录 | 内容 |
|---------------|-----------|------|
| `mysql_data` | Docker 管理 | MySQL 数据文件 |
| `redis_data` | Docker 管理 | Redis 持久化 |
| `milvus_data` | Docker 管理 | Milvus 向量数据 |
| `./uploads` | 项目目录 | 用户上传的知识库文件 |
| `./data/graphs` | 项目目录 | 知识图谱 JSON |
| `./log` | 项目目录 | 应用日志 |

## 5. 常用运维命令

```bash
# 停止所有服务
docker-compose down

# 停止并删除数据卷（⚠️ 清空所有数据）
docker-compose down -v

# 重启单个服务
docker-compose restart backend

# 进入后端容器调试
docker exec -it rag-backend bash

# 查看资源占用
docker stats

# 上传知识库后重建知识图谱
docker exec rag-backend python build_graph.py 6
```

## 6. 服务通信拓扑

```
[用户浏览器]
     │ HTTP :5173
     ▼
[Nginx 前端容器]
     │  /api/* → proxy_pass
     ▼
[FastAPI 后端容器 :8000]
     ├── → [MySQL :3306]       会话/用户/角色数据
     ├── → [Redis :6379]       短期记忆/并发控制
     └── → [Milvus :19530]     向量检索
```

## 7. 生产环境建议

1. **修改 JWT 密钥**：`RAG_JWT_SECRET` 必须设置为随机长字符串
2. **HTTPS**：在 Nginx 中配置 SSL 证书
3. **MySQL 密码**：使用强密码，不要使用默认 `root`
4. **资源限制**：在 docker-compose 中添加 `deploy.resources.limits`
5. **日志轮转**：配置 Docker 日志驱动限制日志大小
6. **备份**：定期备份 MySQL 数据和 Milvus 数据卷
